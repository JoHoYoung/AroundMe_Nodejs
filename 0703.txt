#19. 웹서버 운영하기
 bitnami - 파일 경로 (c드라이브)로 가서, manager-windows를 열면 프로그램을 열 수 있다.
이 프로그램을 이용해서 웹서버를 켜거나 끌 수 있음. 
stop을 누르면 페이지를 볼 수 없게 됨. (어파치 페이지)

설치 후,
bitnami 페이지로 가서 http://127.0.0.1/index.html을 주소창에 입력해도 같은 화면이 출력되는 것을 볼 수 있다.
127.0.0.1 -> 웹 브라우저가 설치되어 있는 곳을 가리키는 특별한 주소. web browser protocol

index.html은 어디에 저장되어 있는 것인가?
c드라이브 -> bitnami 폴더 -> wampstack-7.1.10.1-> apache2 -> ht.docs (http docs)
index라는 파일을 메모장으로 열게 되면 수정이 가능하다.
그리고 그 파일에 있는 문서들을 삭제하고, 우리가 만든 것들을 옮기면 만들었던 홈페이지가 열리는 것을 볼 수 있음.

#그럼, 이렇게 홈페이지 서버를 통해 웹페이지를 제작해서 여는 것과 (http://127.0.0.1/index.html의 주소를 사용), 파일에 들어가서 직접 html을 여는 것은 무슨 차이가 있을까?

1. http://127.0.0.1/index.html
Web browser & Web server
- 웹 브라우저가 웹 서버에게 index.html이라는 페이지를 요청
- 웹 서버가 그 정보를 찾아서 브라우저에게 넘겨준다.
=> http: 서버와 브라우저가 소통할 때 쓰는 것 (Hyper Text 웹페이지 Transfer 주고받기 Protocol 약속, 규정)

2. file://C:/Bitnami/wampstack-7.1.10-1/apache2/htdocs/index.html
Web browser만 존재
- 웹 브라우저가 index.html이라는 페이지를 찾아서 여는 것.

당연히 1번이 질적으로 높을 것!


#두개의 컴퓨터가 웹페이지를 주고받는 방법
1. 웹서버의 주소가 필요.
만약 192.168.0.13이었다면,
브라우저에서는 http://192.168.0.13/index.html 의 주소를 사용해서 웹 서버에게 접근할 수 있다.
-> 스마트폰을 이용할 수 있음!

2. 웹 서버의 주소를 알아내자
와이파이 설정, 나는 192.168.43.186

두개의 창을 띄워보자. (http://127.0.0.1/index.html (자기 자신) http://192.168.43.186/index.html)
같은 와이파이에 접속해야 한다.
이제 다른 컴퓨터에서 저 ip주소를 입력하면 똑같은 홈페이지를 볼 수 있을 것.


=> 서버와 클라이언트, ip주소, 컴퓨터가 서로 통신하는 것들을 알 수 있음
